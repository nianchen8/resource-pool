# 从零到反反爬：新手渐进式指南

> 目标读者：会用 `import requests` 写爬虫，但一跑就被封。

本指南不预设你懂线程、异步、资源池——每一步都是"复制走改改就能跑"的代码，
配合注释告诉你实际发出了什么请求头、为什么这样不会被封。

**统一演示目标**：百度首页 `https://www.baidu.com`。

---

## 0. 前置知识：你的爬虫为什么一跑就封？

新手最常见的反爬误区：**只换 User-Agent**。

```python
# ❌ 这样写，一秒被识别
import requests
requests.get("https://www.baidu.com", headers={"User-Agent": "Mozilla/5.0 ..."})
```

真实浏览器发请求时，至少携带 **5 个关键请求头**（Sendwin 2026 年 Scrapy Fingerprinting 指南[^1] 验证）：

| 请求头 | 浏览器必定发送 | 新手通常 |
|--------|:--:|------|
| `Accept` | ✅ | ❌ 没写 |
| `Accept-Language` | ✅ | ❌ 没写 |
| `Sec-Fetch-Dest` | ✅ | ❌ 不知道有这个 |
| `Sec-Fetch-Mode` | ✅ | ❌ 不知道有这个 |
| `Sec-Ch-Ua` | ✅（Chromium） | ❌ 不知道有这个 |
| `User-Agent` | ✅ | ✅ 只写了这个 |

你只发一个 `User-Agent`，服务器一看："缺了 5 个头，100% 是脚本"——秒封。

**resource-pool 做的事**：你只需 3 行代码，它帮你补全所有请求头，并自动轮换。

[^1]: Sendwin (2026), *Scrapy Fingerprinting: Complete Guide*, https://blog.send.win/scrapy-fingerprinting-complete-guide-2026/

## 1. 单线程 requests（最简入门）

```python
import requests
from user_agent_pool import UserAgentPool

# 创建 UA 池（自动加载 850+ 条真实 UA + 完整请求头）
ua = UserAgentPool()

# 拿到一套完整请求头 → 直接丢给 requests
headers = ua.get_headers("desktop")
resp = requests.get("https://www.baidu.com", headers=headers, timeout=10)
print(resp.status_code)  # 200
```

> **实际发出的请求头**（Chromium 引擎，本次随机到的 UA 是 Chrome 127 on macOS）：
> ```python
> {
>     "User-Agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7) ...",
>     "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,...",
>     "Accept-Encoding": "gzip, deflate, br",
>     "Accept-Language": "zh-CN,zh;q=0.9,en;q=0.8,en-GB;q=0.7,en-US;q=0.6",
>     "Cache-Control": "no-cache",
>     "Connection": "keep-alive",
>     "Sec-Ch-Ua": '"Chromium";v="127", "Not_A Brand";v="99"',
>     "Sec-Ch-Ua-Mobile": "?0",
>     "Sec-Ch-Ua-Platform": '"macOS"',
>     "Sec-Fetch-Dest": "document",
>     "Sec-Fetch-Mode": "navigate",
>     "Sec-Fetch-Site": "none",
>     "Sec-Fetch-User": "?1",
>     "Upgrade-Insecure-Requests": "1",
> }
> # 共 14 个字段，全部覆盖 Sendwin 2026 标准的 5 个关键请求头 ✅
> ```

> 每次 `get_headers()` 返回的字段数量可能不同（4~14 个），取决于本次轮到的 UA 来自 jsonl 还是派系组装。但无论哪种，**同一条 UA 的所有请求头都来自同一个真实设备**，不会出现 Chrome 的 Accept 配 Firefox 的 Accept-Language 这种低级错误。

### 换条 UA 再来一次

```python
# 每次调用自动轮换，不需要手动管理
for _ in range(3):
    headers = ua.get_headers("desktop")
    print(headers["User-Agent"][:50])  # 三次输出三个不同的 UA
```

### 限定移动端

```python
headers = ua.get_headers("mobile")
# → User-Agent 是手机浏览器的，Accept-Language 段数也更少（移动端特征）
```

## 2. 多线程（ThreadPoolExecutor）

并发 10 个请求，每个自动拿到不同的请求头。

```python
import requests
from concurrent.futures import ThreadPoolExecutor, as_completed
from user_agent_pool import UserAgentPool

ua = UserAgentPool()

def fetch(i):
    headers = ua.get_headers("desktop")
    resp = requests.get("https://www.baidu.com", headers=headers, timeout=10)
    return i, resp.status_code

# 10 线程并发
with ThreadPoolExecutor(max_workers=10) as ex:
    futures = [ex.submit(fetch, i) for i in range(10)]
    for f in as_completed(futures):
        i, status = f.result()
        print(f"请求 #{i}: {status}")
```

> 不需要手动加锁——`UserAgentPool` 内置 `ReadWriteLock`，多线程读取自动并发安全。

### 暂存器模式（避免两个线程拿到同一个 UA）

```python
# with ua.reserve() 取出一个 UA → 用完自动归还
with ua.reserve("desktop") as agent:
    # 此时 agent 从池中暂时移除，其他线程拿不到
    requests.get("https://www.baidu.com", headers={"User-Agent": agent})
# 离开 with 块自动归还
```

## 3. 多进程（ProcessPoolExecutor）

每个进程创建独立的池实例。

```python
import requests
from concurrent.futures import ProcessPoolExecutor
from user_agent_pool import UserAgentPool

def fetch(i):
    # ⚠️ 每个进程内独立创建池，不能跨进程共享
    ua = UserAgentPool(thread_safe=False)  # 单进程内可关闭锁
    headers = ua.get_headers("desktop")
    resp = requests.get("https://www.baidu.com", headers=headers, timeout=10)
    return i, resp.status_code

with ProcessPoolExecutor(max_workers=4) as ex:
    for i, status in [f.result() for f in ex.map(fetch, range(8))]:
        print(f"请求 #{i}: {status}")
```

> 多进程场景下，建议每个进程 `thread_safe=False` 消除锁开销。

## 4. 异步 asyncio + aiohttp

```python
import asyncio
import aiohttp
from user_agent_pool.pool_async import AsyncUserAgentPool

async def fetch(session, i):
    headers = await ua.get_headers("desktop")
    async with session.get("https://www.baidu.com", headers=headers) as resp:
        return i, resp.status

async def main():
    global ua
    ua = AsyncUserAgentPool()
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, i) for i in range(10)]
        for i, status in await asyncio.gather(*tasks):
            print(f"请求 #{i}: {status}")

asyncio.run(main())
```

> 同步/异步 API 完全对等——方法名一样，类名前面加 `Async` 就行。

## 5. Scrapy 中间件

`middlewares.py`（放到你的 Scrapy 项目里）：

```python
from user_agent_pool import UserAgentPool

class UAMiddleware:
    def __init__(self):
        self.ua = UserAgentPool()

    @classmethod
    def from_crawler(cls, crawler):
        return cls()

    def process_request(self, request, spider):
        # 每次请求前自动换一套完整的浏览器请求头
        request.headers.update(self.ua.get_headers())
```

`settings.py` 启用：

```python
DOWNLOADER_MIDDLEWARES = {
    "myproject.middlewares.UAMiddleware": 543,
}
```

> Scrapy 默认 UA 是 `Scrapy/2.11.0`，一秒被识别。接入上面 10 行代码后，每次请求自动变成真实浏览器的完整请求头组。
