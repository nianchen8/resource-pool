## 机制一：派系引擎 —— 854 条种子如何变成 14 项请求头

### 全景数据流

```
ua_seeds.json (1 个文件)
    │
    │  _load_unified_seeds()                                   [pool.py:513]
    │  ├─ _header_profiles → 注册到 agents._HEADER_PROFILES
    │  ├─ desktop[] → self._agents["desktop"]
    │  ├─ mobile[]  → self._agents["mobile"]
    │  └─ tablet[]  → self._agents["tablet"]

    │  ── 按"引擎 × 设备"分组 → 每组拆解为 4 种零件
    │
    ▼

    │  _generate_ua_from_faction(browser, os, version)         [pool.py:655]
```

### 数据流一：种子加载

```python
# pool.py:513  —  _load_unified_seeds()
# 1. 打开 user_agent_pool/ua_seeds.json
# 2. 读 _header_profiles → 注册到 agents._HEADER_PROFILES (模块级全局字典)
# 3. 读 desktop/mobile/tablet 三类 → 逐条转 AgentEntry → 追加到 self._agents
```

每条 `AgentEntry`（`pool.py:78` 定义的 `TypedDict`）：

```python
class AgentEntry(TypedDict, total=False):
    ua: str              # 完整的 UA 字符串
    weight: int           # 权重 (1-10)
    profile: str          # Header Profile 名称
    headers: dict         # 直接附加的请求头
    browser: str          # 元数据：chrome | firefox | safari | edge
    os: str               # 元数据：windows | macos | linux | android | ios
    version: int          # 元数据：主版本号 (如 148)
```

`browser` / `os` / `version` 这三个字段**由 `parse_ua_metadata()` 从 UA 字符串自动提取**。随便写一条 Chrome UA 扔进去，池会自动识别它是 Chrome on Windows。

### 数据流二：零件拆解

```python
# pool.py:587  —  _build_ua_component_pools()
# 遍历 self._agents 的每条种子 UA:
#   ① 解析 browser/os/version
#   ② 从 UA 字符串中提取 OS 串 (如 "Windows NT 10.0; Win64; x64")
#   ③ 从 UA 字符串中提取版本令牌 (如 "Chrome/148.0.0.0")
```


**5 大约束自动保证：**

| 约束 | 怎么保证的 |
|------|----------|
| UA 版本 == Sec-Ch-Ua 版本 | `_patch_sec_ch_ua()` 用正则提取 UA 主版本号 → 写回 Sec-Ch-Ua |
| UA 平台 == Sec-Ch-Ua-Platform | 补丁同时修改 Platform 字段 |
| Accept-Language 段数匹配设备类型 | 桌面版 5 段、移动版 3 段 |
| Firefox 不出 Chromium 专属头 | 引擎家族路由——Firefox 分支不创建 Sec-Ch-Ua 等 |
| 引擎家族不交叉 | `match_profile()` 只匹配 browser 同族的 Profile |

## 机制二：ReadWriteLock 写者优先

### 为什么需要读-写分离

UA 池的典型使用模式：

| 操作 | 频率 | 数量级 |
|------|------|--------|
| 读（`get` / `get_headers`） | 每次请求都触发 | 每秒几千次 |
| 写（`add` / `remove`） | 初始化时、手动管理时 | 几乎不发生 |

普通 `threading.Lock` 会让读操作串行化——100 个并发请求 = 100 次锁竞争 = 吞吐量归零。

### 源码关键行（简化版）

```python
# nurture_pool/base.py
class ReadWriteLock:
    def __init__(self):
        self._readers = 0
        self._writer_active = False
        self._writers_waiting = 0
        self._lock = threading.Lock()
        self._no_readers = threading.Condition(self._lock)

    def read_acquire(self):
        with self._lock:
            # ★ 关键判断：有写者在写、或有写者在排队 → 读者等着
            while self._writer_active or self._writers_waiting > 0:
                self._no_readers.wait()
            self._readers += 1

    def write_acquire(self):
        with self._lock:
            self._writers_waiting += 1
            while self._readers > 0 or self._writer_active:
                self._no_readers.wait()
            self._writers_waiting -= 1
            self._writer_active = True
```

### 写者优先的时间线

"写者优先"不是"写者打断正在执行的读者"——是"写者排队时，后来的读者不能插队"。

```
时间线（10 个读者 + 1 个写者）：

t=0ms    读者1~10 开始读（并发进入，_readers=10）
t=5ms    写者请求写锁
         → _writers_waiting = 1
         → while _readers > 0: wait()  ← 等着
         → 新来的读者请求读锁 → while _writers_waiting > 0: wait() ← 排队！
t=10ms   读者1~10 全部读完
         → _readers = 0 → notify_all()
         → 写者被唤醒 → 写操作执行
t=12ms   写者完成
         → _writer_active = False → notify_all()
         → 排队的读者被唤醒 → 并发进入
```

## 机制三：socket patch —— requests 怎么走池内 DNS

### requests 的 DNS 入口

```
requests.get("https://www.baidu.com")
  → urllib3
    → http.client
      → socket.create_connection()
        → socket.getaddrinfo("www.baidu.com", 443)
          → 系统 DNS
```

`requests` 没有参数可以"指定 DNS 服务器"——唯一的办法是 monkey-patch：**运行时替换 `socket.getaddrinfo`**。

### 源码关键行

```python
# dns_resolver_pool/pool.py

class DNSResolverPool:
    def __init__(self, ...):
        self._socket_patched = False
        self._original_getaddrinfo = None

    def patch_socket(self):
        if self._socket_patched:
            return
        self._original_getaddrinfo = socket.getaddrinfo   # ★ 保存原版
        socket.getaddrinfo = self._patched_getaddrinfo     # ★ 全局替换
        self._socket_patched = True

    def unpatch_socket(self):
        if self._original_getaddrinfo is not None:
            socket.getaddrinfo = self._original_getaddrinfo  # ★ 恢复
            self._original_getaddrinfo = None
            self._socket_patched = False

    def _patched_getaddrinfo(self, host, port, family=0, type=0, proto=0, flags=0):
        # 1. 域名才走池内 DNS；IP 直接系统 DNS
        if self._is_ip_address(host):
            return self._original_getaddrinfo(host, port, ...)

        # 2. 走池内 DNS
        try:
            ips = self.resolve_all(host, timeout=5.0)
        except Exception:
            if self._fallback_to_system:
                return self._original_getaddrinfo(host, port, ...)
            raise

        # 3. 把 IP 列表转回 getaddrinfo 标准格式
        return [(family, type, proto, '', (ip, port)) for ip in ips]
```

### with 语句的魔法

```python
with dns:      # dns.__enter__() → patch_socket()
    ...         # 所有 socket.getaddrinfo 都走池内 DNS
                # dns.__exit__() → unpatch_socket()
```

socket patch **只在 with 块内生效**。退出后自动恢复。

### 🔍 探查脚本：socket patch 探针

```python
import socket, nurture_pool
from concurrent.futures import ProcessPoolExecutor

def inspect_patch(i):
    dns = nurture_pool.DNS()

    before_id = id(socket.getaddrinfo)
    print(f"[子进程{i}] patch 前: getaddrinfo id={before_id}")

    dns.__enter__()

    after_id = id(socket.getaddrinfo)
    print(f"[子进程{i}] patch 后: getaddrinfo id={after_id} "
          f"({'已替换' if before_id != after_id else '未变化'})")

    ips = socket.getaddrinfo("www.baidu.com", 443)
    print(f"[子进程{i}] 解析结果: {len(ips)} 条地址记录")

    dns.__exit__(None, None, None)

    restored_id = id(socket.getaddrinfo)
    print(f"[子进程{i}] unpatch 后: getaddrinfo id={restored_id} "
          f"({'已恢复' if restored_id == before_id else '未恢复'})")

    return i

if __name__ == '__main__':
    with ProcessPoolExecutor(max_workers=2) as ex:
        for i, _ in enumerate(ex.map(inspect_patch, range(2))):
            pass
```

## 机制四：dispatch 注册表 —— 编排器怎么知道调哪个方法

### 两级分派

`PoolOrchestrator` 接收任意 `**extra_pools`。它怎么知道遇到 `CookiePool` 要调 `.pick()`？

答案：**`isinstance` 精确匹配 + `hasattr` 兜底遍历**。

### 源码关键行

```python
# nurture_pool/orchestrator.py

class PoolOrchestrator:
    _DISPATCH: dict[type, str] = {}    # ★ 类级别注册表

    @classmethod
    def register_dispatch(cls, pool_type: type, method_name: str) -> None:
        cls._DISPATCH[pool_type] = method_name

    def _fetch_from_pool(self, pool) -> Any:
        # ── 第一级：isinstance 精确匹配（注册表）──
        for pool_type, method_name in self._DISPATCH.items():
            if isinstance(pool, pool_type):
                method = getattr(pool, method_name)
                if asyncio.iscoroutinefunction(method):
                    return method
                return method()

        # ── 第二级：hasattr 兜底 ──
        for attr in ("get_headers", "get", "get_dict", "resolve", "pick"):
            if hasattr(pool, attr):
                method = getattr(pool, attr)
                if asyncio.iscoroutinefunction(method):
                    return method
                return method()

        raise TypeError(f"无法为 {type(pool).__name__} 确定资源获取方法")
```

### 内置注册时机

三个池的 `register_dispatch` 写在各池的 `__init__.py` 中——**只要 `import` 了池模块，注册自动完成**。

```python
# user_agent_pool/__init__.py
PoolOrchestrator.register_dispatch(UserAgentPool, "get_headers")

# dns_resolver_pool/__init__.py
PoolOrchestrator.register_dispatch(DNSResolverPool, "get_server")

# proxy_pool/__init__.py
PoolOrchestrator.register_dispatch(ProxyPool, "get_dict")
```

## 机制五：ABC 基类设计

### ResourcePool 基类

```python
# nurture_pool/base.py

class ResourcePool(ABC):
    """所有资源池的抽象基类——定义资源池必须实现的 5 个方法。
    内置池并未直接继承它，但按此契约实现了对应方法。"""

    @abstractmethod
    def get(self, *args, **kwargs) -> Any: ...
    @abstractmethod
    def add(self, *args, **kwargs) -> None: ...
    @abstractmethod
    def remove(self, *args, **kwargs) -> bool: ...
    @abstractmethod
    def health_check(self, **kwargs) -> Any: ...
    @abstractmethod
    def stats(self) -> Any: ...
```

### 为什么内置池没继承它

`UserAgentPool.get(category="desktop")`、`DNSResolverPool.resolve(domain)`、`ProxyPool.get(scheme=None)` — 方法签名各不相同。强制继承会失去灵活性。所以 ABC 做**文档和契约参考**。

## 机制六：短别名惰性加载

```python
# nurture_pool/_shortcuts.py

def combo(**pools):
    """一行组合多池资源→返回 PoolCombo"""
    from nurture_pool.orchestrator import PoolOrchestrator
    inner = {}
    for name, p in pools.items():
        if hasattr(p, '_pool') and hasattr(p, '_ensure'):
            p._ensure()
            inner[name] = p._pool
        else:
            inner[name] = p
    return PoolOrchestrator(**inner).next()

def DNS():
    from dns_resolver_pool import DNSResolverPool
    return DNSResolverPool()
```

**惰性导入**——`combo()` 里的 `import` 只在调用时执行。避免冷启动时加载 `ua_seeds.json`。

短别名通过 `__init__.py` 的 `__getattr__` 惰性加载机制对外暴露：
```python
# nurture_pool/__init__.py
_LAZY_IMPORTS = {
    "UA":    ("nurture_pool._shortcuts", "UA"),
    "Proxy": ("nurture_pool._shortcuts", "Proxy"),
    "DNS":   ("nurture_pool._shortcuts", "DNS"),
    "combo": ("nurture_pool._shortcuts", "combo"),
}

def __getattr__(name):
    if name in _LAZY_IMPORTS:
        module_path, attr = _LAZY_IMPORTS[name]
        mod = importlib.import_module(module_path)
        value = getattr(mod, attr)
        globals()[name] = value  # 缓存，避免重复加载
        return value
    raise AttributeError(...)
```

## Scrapy 中间件：dispatch 全链路诊断

```python
import requests
from user_agent_pool import UserAgentPool
from dns_resolver_pool import DNSResolverPool
from nurture_pool import PoolOrchestrator

ua = UserAgentPool()
dns = DNSResolverPool()

print("=== 分派诊断 ===")
print(f"注册表: {list(PoolOrchestrator._DISPATCH.keys())}")

orch = PoolOrchestrator(ua=ua, dns=dns)

dns.__enter__()
c = orch.next()
# 编排器内部：
#   isinstance(ua, UserAgentPool)   → True  → 调 .get_headers()
#   isinstance(dns, DNSResolverPool) → True  → 调 .get_server()

print(f"c.ua 类型: {type(c.ua).__name__}, 字段数: {len(c.ua)}")
print(f"c.dns: {c.dns}")

resp = requests.get("https://www.baidu.com",
                    headers=c.ua, timeout=10)
print(f"状态码: {resp.status_code}")

dns.__exit__(None, None, None)

print("\n=== 反爬关键字段 ===")
for key in ["User-Agent", "Accept", "Accept-Language",
            "Sec-Ch-Ua", "Sec-Ch-Ua-Platform"]:
    print(f"  {key}: {c.ua.get(key, '(缺失)')[:80]}")
```
