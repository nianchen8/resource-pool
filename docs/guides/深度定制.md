# 深度定制

> 你已经会拧旋钮了（初级定制）。现在的问题是：**如果内置能力不够，你怎么自己造？**

这篇覆盖三个话题，每个都从"是什么 → 怎么用 → 为什么 → 踩什么坑"四个角度讲透：

1. **自定义策略**：把内置枚举换成你自己的选择规则
2. **自定义池**：写一个 Cookie 池/Token 池——接入编排器，享受故障隔离
3. **锁机理**：理解三种锁，避免你的自定义池引入死锁

---

## 话题一：自定义策略

### 内置枚举的局限

初级定制的旋钮中，三个策略枚举各只有 2~3 个选项：

| 池 | 策略枚举 | 选项 |
|---|--------|------|
| UA | `UAStrategy` | `WEIGHTED`（加权）、`UNIFORM`（均匀） |
| DNS | `SelectStrategy` | `LATENCY_WEIGHTED`、`ROUND_ROBIN`、`RANDOM` |
| Proxy | `ProxyStrategy` | `LATENCY_WEIGHTED`、`ROUND_ROBIN`、`RANDOM` |

问题来了：**如果你只想要 Chrome 120 以上版本的 UA？如果你想让代理按地理位置优先（国内优先于海外）？如果你想让 DNS 选择避开某个 IP 段？** ——枚举不够用。

### `StrategyProtocol` 协议

内置池的 `strategy` 参数接受两种类型：

```
strategy: UAStrategy | StrategyProtocol
```

其中 `StrategyProtocol` 的定义是：

```python
# resource_pool/base.py
from typing import Protocol, Iterator, TypeVar

T = TypeVar("T")

class StrategyProtocol(Protocol[T]):
    """自定义策略协议

    你的 callable 实例必须满足：
    - 接收 list[T]（候选资源列表）
    - 返回 Iterator[T]（一次 yield 一个被选中的资源）
    """
    def __call__(self, items: list[T]) -> Iterator[T]: ...
```

**关键约束：**

1. **必须是可调用对象**（实现了 `__call__` 的类实例、或普通函数、或 lambda）
2. **接收 `list[T]`** ——池会把所有候选资源传给你的 callable
3. **返回 `Iterator[T]`** ——`yield` 一个出来，而不是 `return`

> ⚠️ **为什么 `yield` 而不是 `return`？** `yield` 返回迭代器——它"懒"：调用者拿到迭代器后只需 `next()` 一次就拿到结果，不需要跑完所有逻辑。更重要的是——用 `yield` 可以在你不选任何资源时触发 `StopIteration`，让编排器感知"这个池枯竭了"。

### 自定义策略示例（3 个场景）

**场景 1：只挑 Chrome 120+ 的 UA**

```python
class PickLatestChrome:
    """自定义策略：只在 Chrome 120+ 的 UA 中选"""
    def __call__(self, items):
        # items 是 list[AgentEntry]，每个 entry 有 browser/os/version 等字段
        chrome = [i for i in items
                  if getattr(i, "browser", "") == "chrome"
                  and getattr(i, "version", 0) >= 120]
        if not chrome:
            chrome = items              # ★ 找不到时回退到全部——避免抛异常
        yield random.choice(chrome)

ua = UserAgentPool(strategy=PickLatestChrome())
```

**场景 2：代理按地理位置优先（国内 → 海外）**

```python
class DomesticFirst:
    """自定义策略：国内代理优先，没有才用海外"""
    def __call__(self, items):
        domestic = [p for p in items if getattr(p, "region", "") == "china"]
        if domestic:
            yield min(domestic, key=lambda p: p.latency_ms)
        else:
            yield min(items, key=lambda p: p.latency_ms)

proxy = ProxyPool(strategy=DomesticFirst())
```

**场景 3：DNS 避开某 IP 段**

```python
class AvoidIPRange:
    """自定义策略：避开 8.8.*.* 网段（Google DNS 有时被 QOS）"""
    def __call__(self, items):
        safe = [s for s in items if not s.ip.startswith("8.8.")]
        yield random.choice(safe if safe else items)

dns = DNSResolverPool(strategy=AvoidIPRange())
```

### ⚠️ 锁内执行：策略 callable 的禁忌

**你的策略 callable 在锁内执行。** 调用链：

```
orch.next()
  → ua.get_headers()
    → with self._lock.read():      # ← 这里已经持有读锁
        → strategy(candidates)     # ← 你的代码在这里执行
```

**绝对不能做的事：**

| 不要 | 原因 | 后果 |
|------|------|------|
| 不要做 HTTP 请求 | 在锁内做 IO | 所有线程在锁外排队，吞吐量归零 |
| 不要 `time.sleep()` | 在锁内阻塞 | 同上——锁被占着不放 |
| 不要调其他池的方法 | 可能触发另一个锁 | 死锁风险 |
| 不要修改 `items` 列表 | 改共享数据 | 数据竞争，后续调用结果不确定 |

**可以做的事：**
- 过滤（`[i for i in items if ...]`）
- 排序（`sorted(items, key=...)`）
- 随机选（`random.choice(items)`）
- 拿对象属性（`i.browser`、`i.version`、`i.latency_ms`）

## 话题二：自定义池

### 为什么需要自定义池

内置三件套（UA + DNS + Proxy）覆盖了最常见的反反爬需求。但生产环境你可能还需要：

- **Cookie 池**：1000 个已登录 Cookie，轮流换
- **Token 池**：Bearer Token 自动刷新
- **指纹池**：Canvas/WebGL 指纹切换
- **IP 池**：除了代理之外的其他 IP 管理方式

这些你自己维护、自己更新的资源，需要接入编排器——让你可以 `orch.next()` 一次性拿到 UA + DNS + Cookie，而不是单独管理多个池。

### `register_dispatch` 机制

编排器维护一个**类级别的分派注册表** `_DISPATCH`：

```python
# 编排器内部（简化版）
class PoolOrchestrator:
    _DISPATCH: dict[type, str] = {}    # {池类: 方法名}

    @classmethod
    def register_dispatch(cls, pool_type, method_name):
        cls._DISPATCH[pool_type] = method_name

    def _fetch_from_pool(self, pool):
        # 1. isinstance 精确匹配（优先级最高）
        for pool_type, method_name in self._DISPATCH.items():
            if isinstance(pool, pool_type):
                return getattr(pool, method_name)()

        # 2. hasattr 兜底
        for attr in ("get_headers", "get", "get_dict", "resolve", "pick"):
            if hasattr(pool, attr):
                return getattr(pool, attr)()

        raise TypeError(f"无法为 {type(pool).__name__} 确定资源获取方法")
```

**遍历顺序决定优先级：**

1. 先遍历 `_DISPATCH` 注册表——`isinstance` 匹配（精确、确定性强）
2. 注册表没命中 → `hasattr` 兜底，依次试 `get_headers` → `get` → `get_dict` → `resolve` → `pick`

**内置注册表（自动注册）：**

| 池类型 | 调的方法 | 注册时机 |
|--------|---------|---------|
| `UserAgentPool` | `.get_headers()` | 导入 `user_agent_pool` 时 |
| `DNSResolverPool` | `.get_server()` | 导入 `dns_resolver_pool` 时 |
| `ProxyPool` | `.get_dict()` | 导入 `proxy_pool` 时 |

### 自定义池的最小契约

要实现一个能接入编排器的自定义池，你的类只需要：

1. **`__len__`** — 返回可用资源数量（编排器做统计用）
2. **`__repr__`** — 可读的字符串表示（日志用）
3. **一个取值方法**（如 `pick()`）——调用 `register_dispatch` 注册后，编排器就知道要调它

```python
class CookiePool:
    def __init__(self, cookies: list[str]):
        self._cookies = cookies[:]

    def __len__(self):           # ★ 必须
        return len(self._cookies)

    def __repr__(self):          # ★ 必须
        return f"CookiePool({len(self)})"

    def pick(self) -> str:       # ★ 你的取值方法
        return random.choice(self._cookies)

# 注册：告诉编排器"遇到 CookiePool 调 .pick()"
PoolOrchestrator.register_dispatch(CookiePool, "pick")
```

> ℹ️ 不注册也行——因为 `hasattr` 兜底会遍历 `pick` 方法名。但**强烈建议显式注册**：你的意图更清晰，不会因为未来方法名冲突导致意外行为。

### 进阶：带故障隔离的自定义池

简单的 Cookie 池够用了——但如果某个 Cookie 过期了（请求返回 403），你希望**自动标记、自动隔离、自动复活**。

和内置的 DNS/Proxy 池一样的机制——你需要自己实现以下状态机：

```
[已启用] ──fail──→ [fail_count+=1] ──fail_count >= max_fails──→ [已隔离]
[已隔离] ──revive_after 到期──→ [已启用]（试用复活）
[已隔离] ──试用失败──→ [fail_count += 1] ──→ [已隔离]（重新倒计时）
```

**完整实现：`RobustCookiePool`**

```python
class RobustCookiePool:
    """带故障隔离的 Cookie 池

    隔离时间线：
    14:00:00  第1次失败 → fail_count=1
    14:00:05  第2次失败 → fail_count=2
    14:00:10  第3次失败 → fail_count=3 ≥ max_fails → 隔离
    14:02:10  revive_after 到期 → 复活 → 下次请求再试用
    """
    def __init__(self, cookies: list[str], max_fails=3, revive_after=120):
        self._items = [
            {"value": c, "fails": 0, "enabled": True, "isolated_at": 0.0}
            for c in cookies
        ]
        self._lock = threading.Lock()
        self._max_fails = max_fails
        self._revive_after = revive_after

    def __len__(self):
        return sum(1 for i in self._items if i["enabled"])

    def __repr__(self):
        return f"RobustCookiePool(alive={len(self)}/{len(self._items)})"

    def pick(self) -> str:
        with self._lock:
            self._try_revive()                # ★ 先复活到期的
            alive = [i for i in self._items if i["enabled"]]
            if not alive:
                raise PoolExhaustedError("all cookies dead")
            return random.choice(alive)["value"]

    def mark_failed(self, cookie_value: str):
        """标记一个 Cookie 失败（使用者在外层调）"""
        with self._lock:
            for item in self._items:
                if item["value"] == cookie_value:
                    item["fails"] += 1
                    if item["fails"] >= self._max_fails:
                        item["enabled"] = False
                        item["isolated_at"] = time.time()
                    break

    def _try_revive(self):
        """复活隔离时间到期的 Cookie"""
        now = time.time()
        for item in self._items:
            if not item["enabled"] and \
               (now - item["isolated_at"]) > self._revive_after:
                item["enabled"] = True
                item["fails"] = max(0, item["fails"] - 1)  # 给一次机会
```

> 💡 **关键设计：失败计数不清零**。复活后的 Cookie 只有 `fails - 1`，不是清零。这样如果它复活后立刻又失败，会更快地被重新隔离——避免"复活 → 失败 → 复活 → 失败"的抖动。

### 自定义池的变体：Token 池（带自动刷新）

```python
class TokenPool:
    """Token 池：拿一个 Bearer Token，支持自动刷新"""
    def __init__(self, tokens: list[str], refresh_url: str, ttl: int = 1800):
        self._tokens = [{"value": t, "expires_at": time.time() + ttl}
                        for t in tokens]
        self._refresh_url = refresh_url
        self._lock = threading.Lock()

    def __len__(self):
        return len(self._tokens)

    def __repr__(self):
        return f"TokenPool({len(self)})"

    def pick(self) -> str:
        with self._lock:
            fresh = [t for t in self._tokens
                     if t["expires_at"] > time.time() + 60]
            if not fresh:
                self._refresh_all()
                fresh = self._tokens
            return random.choice(fresh)["value"]

    def _refresh_all(self):
        # ⚠️ 不要在这里做 HTTP 请求！它在锁内。
        # 正确做法：锁外刷新，然后用 add_token 加进来。
        pass

PoolOrchestrator.register_dispatch(TokenPool, "pick")
```

## 话题三：锁机理

### 三种锁

| 锁 | 位置 | 并发模式 | 开销 | 可重入 |
|---|------|---------|------|-------|
| `ReadWriteLock` | `resource_pool.base` | 读共享、写独占、写者优先 | 低 | 否 |
| `threading.Lock` | Python 标准库 | 互斥，一次一个线程 | 很低 | 否 |
| `asyncio.Lock` | Python 标准库 | 协程互斥，一次一个协程 | 极低 | 否 |

### `ReadWriteLock` 详解（UA 池用）

**读写场景：**
- 读操作（`get`/`get_headers`）：**极高频**，每次请求都触发
- 写操作（`add`/`remove`/`load`）：**低频**，只在初始化或手动管理时触发

`ReadWriteLock` 允许 N 个读操作同时进入——**这是 UA 池吞吐量 ~111k ops/s 的核心原因**。

**写者优先：**当有写者在排队时，新读者不能进入——防止高频读操作把写者永远饿死。

### 什么时候用哪种锁

| 你的场景 | 用哪种锁 | 理由 |
|----------|---------|------|
| 频繁读、偶尔写 | `ReadWriteLock` | 读并发，最大化吞吐量 |
| 读写都频繁，操作极快 | `threading.Lock` | 简单、开销最低 |
| 异步协程 | `asyncio.Lock` | 协程安全，不阻塞事件循环 |
| 单线程环境 | 任何 `Dummy*` | 空操作，零开销 |

### 死锁防范：三条铁律

1. **不要在锁内做 IO** ——不光是你的代码，调用的函数里也不要有 `requests.get()`、`time.sleep()`。
2. **不要锁内调另一个锁的方法** ——比如在 `pick()` 里（持有锁）调用 `proxy.get()`（它也加锁）。
3. **异步锁不可重入** ——`asyncio.Lock` 同一个协程不能两次 `await lock.acquire()`。**一个方法只加一次锁。**

## 单线程

和初级定制的区别：**内置枚举不够用——用自定义 callable 控制选取逻辑，再加一个自己的 Cookie 池挂到编排器上**。

```python
import requests, random
from user_agent_pool import UserAgentPool
from dns_resolver_pool import DNSResolverPool
from resource_pool import PoolOrchestrator
from resource_pool.exceptions import PoolExhaustedError

# ── ① 自定义策略：只挑 Chrome 120+ 的 UA ──
class PickLatestChrome:
    """策略 callable：接收候选 UA 列表，yield 一个被选中的"""
    def __call__(self, items):
        chrome = [i for i in items
                  if getattr(i, "browser", "") == "chrome"
                  and getattr(i, "version", 0) >= 120]
        if not chrome:
            chrome = items                # ★ 找不到就全量退回——安全阀
        yield random.choice(chrome)

ua = UserAgentPool(strategy=PickLatestChrome())

# ── ② 自定义 Cookie 池 ──
class CookiePool:
    """最简 Cookie 池：实现 __len__/__repr__/pick 三个方法即可"""
    def __init__(self, cookies: list[str]):
        self._cookies = cookies[:]

    def __len__(self):                    # ★ 编排器做统计用
        return len(self._cookies)

    def __repr__(self):                   # ★ 日志可读
        return f"CookiePool({len(self)})"

    def pick(self) -> str:                # ★ 取值方法
        return random.choice(self._cookies)

# ── ③ 注册分派 ──
PoolOrchestrator.register_dispatch(CookiePool, "pick")

# ── ④ 用起来 ──
dns = DNSResolverPool()
cookie = CookiePool(["token=abc123", "token=def456", "token=ghi789"])
orch = PoolOrchestrator(ua=ua, dns=dns, cookie=cookie)

with dns:
    c = orch.next()
    # c.ua     → Chrome 120+ 的完整请求头（14 项）
    # c.cookie → "token=abc123"（automatically via cookie.pick()）
    resp = requests.get("https://www.baidu.com",
                        headers=c.ua,
                        cookies={"Cookie": c.cookie},
                        timeout=10)
print(resp.status_code)
```

**逐行讲解**

1. `PickLatestChrome.__call__` 接收 `list[AgentEntry]`。每个 entry 有 `browser`、`os`、`version` 等字段—用 `getattr` 安全取值。

2. `if not chrome: chrome = items` ——安全阀。筛选条件太严格导致候选集为空时，退回全部候选。

3. `yield` 不是 `return` ——返回迭代器，惰性执行。调用者 `next()` 拿到第一个结果就停止。

4. `PoolOrchestrator.register_dispatch(CookiePool, "pick")` ——只影响同步版编排器。异步版需另外注册 `AsyncPoolOrchestrator.register_dispatch(...)`。

5. keyword argument 的名字 `cookie` 决定了 `PoolCombo` 上的属性名 `c.cookie`。

## 多线程

和初级定制的区别：**带故障隔离的自定义池 + 后台健康检查线程**。

```python
import requests, random, threading, time
from concurrent.futures import ThreadPoolExecutor, as_completed
from user_agent_pool import UserAgentPool
from dns_resolver_pool import DNSResolverPool
from resource_pool import PoolOrchestrator
from resource_pool.exceptions import PoolExhaustedError

# ── ① 带故障隔离的 Cookie 池 ──
class RobustCookiePool:
    def __init__(self, cookies: list[str], max_fails=3, revive_after=120):
        self._items = [
            {"value": c, "fails": 0, "enabled": True, "isolated_at": 0.0}
            for c in cookies
        ]
        self._lock = threading.Lock()
        self._max_fails = max_fails
        self._revive_after = revive_after

    def __len__(self):
        return sum(1 for i in self._items if i["enabled"])

    def __repr__(self):
        return f"RobustCookiePool(alive={len(self)}/{len(self._items)})"

    def pick(self) -> str:
        with self._lock:
            self._try_revive()                # 先复活到期的
            alive = [i for i in self._items if i["enabled"]]
            if not alive:
                raise PoolExhaustedError("all cookies dead")
            return random.choice(alive)["value"]

    def mark_failed(self, cookie_value: str):
        """标记 Cookie 失败；达到阈值自动隔离"""
        with self._lock:
            for item in self._items:
                if item["value"] == cookie_value:
                    item["fails"] += 1
                    if item["fails"] >= self._max_fails:
                        item["enabled"] = False
                        item["isolated_at"] = time.time()
                    break

    def _try_revive(self):
        """复活隔离到期的 Cookie"""
        now = time.time()
        for item in self._items:
            if not item["enabled"] and \
               (now - item["isolated_at"]) > self._revive_after:
                item["enabled"] = True
                item["fails"] = max(0, item["fails"] - 1)

PoolOrchestrator.register_dispatch(RobustCookiePool, "pick")

# ── ② 三件套 + 自定义池 ──
class PickLatestChrome:
    def __call__(self, items):
        chrome = [i for i in items
                  if getattr(i, "browser", "") == "chrome"
                  and getattr(i, "version", 0) >= 120]
        if not chrome:
            chrome = items
        yield random.choice(chrome)

ua = UserAgentPool(strategy=PickLatestChrome())
dns = DNSResolverPool()
cookie = RobustCookiePool(
    ["token=aa", "token=bb", "token=cc", "token=dd", "token=ee"],
    max_fails=3,
    revive_after=120,
)
orch = PoolOrchestrator(ua=ua, dns=dns, cookie=cookie)

# ── ③ 后台健康检查线程（生产环境标配）──
def health_loop(pools, interval=300):
    while True:
        for p in pools:
            if hasattr(p, "health_check"):
                p.health_check(timeout=10)
            elif hasattr(p, "_try_revive"):
                with p._lock:
                    p._try_revive()
        time.sleep(interval)

threading.Thread(target=health_loop, args=([dns, cookie],),
                 daemon=True).start()

# ── ④ 多线程跑 ──
with dns:
    def fetch(i):
        try:
            c = orch.next()
            resp = requests.get("https://www.baidu.com",
                                headers=c.ua,
                                cookies={"Cookie": c.cookie},
                                timeout=10)
            return i, resp.status_code
        except PoolExhaustedError:
            return i, "池枯竭"

    with ThreadPoolExecutor(max_workers=10) as ex:
        for f in as_completed([ex.submit(fetch, i) for i in range(10)]):
            i, status = f.result()
            print(f"#{i}: {status}")
```

**逐行讲解**

1. `RobustCookiePool` 每个 Cookie 有四个字段：`value`、`fails`、`enabled`、`isolated_at`。

2. `pick()` 里先调 `_try_revive()` ——每次取 Cookie 时检查有没有可以复活的。复活是**懒触发**的。

3. `mark_failed()` 需要在请求失败时由使用方主动调用。

4. 后台线程 `health_loop` 每 5 分钟跑一次。`daemon=True` 让它在主线程退出时自动终止。

## 多进程

和初级定制的区别：**每子进程内建自定义池 + 注册分派**。

```python
import requests, random
from concurrent.futures import ProcessPoolExecutor
from user_agent_pool import UserAgentPool
from dns_resolver_pool import DNSResolverPool
from resource_pool import PoolOrchestrator

# ── 自定义池定义在模块级（子进程可 pickle）──
class CookiePool:
    def __init__(self, cookies):
        self._cookies = list(cookies)
    def __len__(self):
        return len(self._cookies)
    def __repr__(self):
        return f"CookiePool({len(self)})"
    def pick(self):
        return random.choice(self._cookies)

PoolOrchestrator.register_dispatch(CookiePool, "pick")

class PickLatestChrome:
    def __call__(self, items):
        chrome = [i for i in items
                  if getattr(i, "browser", "") == "chrome"
                  and getattr(i, "version", 0) >= 120]
        yield random.choice(chrome or items)

def fetch(i):
    ua = UserAgentPool(strategy=PickLatestChrome(), thread_safe=False)
    dns = DNSResolverPool(thread_safe=False)
    cookie = CookiePool([f"token_proc_{i}_aa", f"token_proc_{i}_bb"])
    orch = PoolOrchestrator(ua=ua, dns=dns, cookie=cookie)

    with dns:
        c = orch.next()
        resp = requests.get("https://www.baidu.com",
                            headers=c.ua,
                            cookies={"Cookie": c.cookie},
                            timeout=10)
    return i, resp.status_code

if __name__ == '__main__':
    with ProcessPoolExecutor(max_workers=4) as ex:
        for i, status in [f.result() for f in ex.map(fetch, range(8))]:
            print(f"#{i}: {status}")
```

**逐行讲解**

1. **模块级定义是强制的** ——类定义在函数内会导致 `pickle` 失败。

2. `thread_safe=False` ——子进程内单线程，所有锁变空操作。自定义池没加锁——只有一个人在跑。

3. 子进程之间不共享状态——每进程独立 `CookiePool` 实例。

## 异步

和初级定制的区别：**异步版的自定义策略 + 自定义池**。

```python
import asyncio, aiohttp, random
from user_agent_pool.pool_async import AsyncUserAgentPool
from dns_resolver_pool.pool_async import AsyncDNSResolverPool
from resource_pool.orchestrator_async import AsyncPoolOrchestrator
from resource_pool.exceptions import PoolExhaustedError

# ── 异步版 Cookie 池 ──
class AsyncCookiePool:
    def __init__(self, cookies: list[str]):
        self._cookies = cookies[:]
        self._lock = asyncio.Lock()          # ★ asyncio.Lock

    def __len__(self):
        return len(self._cookies)

    def __repr__(self):
        return f"AsyncCookiePool({len(self)})"

    async def pick(self) -> str:             # ★ async 方法
        async with self._lock:               # ★ async with
            return random.choice(self._cookies)

AsyncPoolOrchestrator.register_dispatch(AsyncCookiePool, "pick")

# ── 异步版自定义策略（和同步版完全一样）──
class PickLatestChrome:
    def __call__(self, items):
        chrome = [i for i in items
                  if getattr(i, "browser", "") == "chrome"
                  and getattr(i, "version", 0) >= 120]
        yield random.choice(chrome or items)

ua = AsyncUserAgentPool(strategy=PickLatestChrome())
dns = AsyncDNSResolverPool()
cookie = AsyncCookiePool(["token=aa", "token=bb", "token=cc"])
orch = AsyncPoolOrchestrator(ua=ua, dns=dns, cookie=cookie)

async def fetch(session, i):
    c = await orch.next()
    async with session.get("https://www.baidu.com",
                           headers=c.ua,
                           cookies={"Cookie": c.cookie}) as resp:
        return i, resp.status

async def main():
    connector = aiohttp.TCPConnector(resolver=dns.create_resolver())
    async with aiohttp.ClientSession(connector=connector) as session:
        for i, status in await asyncio.gather(
            *[fetch(session, i) for i in range(10)]
        ):
            print(f"#{i}: {status}")

asyncio.run(main())
```

**逐行讲解**

1. 异步池用 `asyncio.Lock` ——不阻塞事件循环；等待时让出控制权给其他协程。

2. 取值方法加 `async` ——`pick()` 变成 `async def pick()`。编排器内置对 async 方法的支持。

3. 注册表独立 ——`AsyncPoolOrchestrator.register_dispatch(...)` 和同步版是两个不同的注册表。

4. 自定义策略不需要改 ——`PickLatestChrome` 和同步版完全一样。策略 callable 是同步的。

5. ⚠️ **异步锁不可重入** ——一个方法只加一次锁。

```python
import random
from user_agent_pool import UserAgentPool
from dns_resolver_pool import DNSResolverPool
from resource_pool import PoolOrchestrator

class PickLatestChrome:
    def __call__(self, items):
        chrome = [i for i in items
                  if getattr(i, "browser", "") == "chrome"
                  and getattr(i, "version", 0) >= 120]
        yield random.choice(chrome or items)

class CookiePool:
    def __init__(self, cookies):
        self._cookies = list(cookies)
    def __len__(self):
        return len(self._cookies)
    def __repr__(self):
        return f"CookiePool({len(self)})"
    def pick(self):
        return random.choice(self._cookies)

PoolOrchestrator.register_dispatch(CookiePool, "pick")

class DeepCustomMiddleware:
    def __init__(self):
        self.ua = UserAgentPool(strategy=PickLatestChrome())
        self.dns = DNSResolverPool()
        self.cookie = CookiePool(["token=scrapy1", "token=scrapy2"])
        self.orch = PoolOrchestrator(ua=self.ua, dns=self.dns,
                                      cookie=self.cookie)

    def process_request(self, request, spider):
        self.dns.__enter__()
        c = self.orch.next()
        request.headers.update(c.ua)
        if c.cookie:
            request.headers["Cookie"] = c.cookie

    def process_response(self, request, response, spider):
        self.dns.__exit__(None, None, None)
        return response

    def process_exception(self, request, exception, spider):
        self.dns.__exit__(None, None, None)
```

## 跑不通？看这里

| 现象 | 原因 | 解法 |
|------|------|------|
| 策略 callable 内做了 HTTP 请求 | 在锁内 IO | 把筛选逻辑和数据获取分离——先锁外拿数据，再传进 callable |
| `return` 了而不是 `yield` | 类型签名不匹配 | 改成 `yield`——策略协议要求返回 Iterator |
| 多进程 `Can't pickle` | 自定义类定义在函数内 | 把所有自定义类定义移到模块级 |
| `AsyncPoolOrchestrator` 不认识自定义池 | 注册到同步版了 | `AsyncPoolOrchestrator.register_dispatch(...)` 是独立的 |
| 异步 `pick()` 死锁 | `asyncio.Lock` 不可重入 | 一个方法只加一次锁 |
| Cookie 池全部枯竭不恢复 | 没有 revive 逻辑 | 参考 `RobustCookiePool._try_revive()` |
